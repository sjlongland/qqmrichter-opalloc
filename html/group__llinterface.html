Detailed Description

This is the nuts-and-bolts interface to the library. It is primarily used to implement the high level interface and the macro interface but has been exposed to enable end-users to make task-specific interfaces if they so choose.

It is strongly recommended not to use this interface directly as it is highly error-prone.

op_allocator op_ll_initialize_allocator ( const size_t  object_size,
const size_t  initial_count,
const op_ll_allocator_mode  mode 
)

Initialize an allocator according to the provided configuration.

Parameters
[in]object_sizeThe size each individual object requires in RAM.
[in]initial_countThe starting size of the allocator's array.
[in]use_chunksSignals this allocator uses chunk allocation.
[in]use_linearSignals this allocator uses linear growth.
Returns
An op_allocator handle used in subsequent operations or NULL on failure.
Note
1. If use_chunks is not provided, only the indirect array is calloced in the allocator. If use_chunks is provided, the first initial_count objects of object_size bytes are also allocated.
  1. On failure NULL is returned, but op_error_handler() is called first.
